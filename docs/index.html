ClaudeBar drives the claude binary you already have installed — no bundling. Visual session switching, inline tool-call pills and approval prompts, markdown chat with syntax highlighting, and a slide-out operator panel for plugins, skills, commands, and token stats. All in a 400×800 draggable window that lives in your menu bar.

An optional mascot, tethered to your desktop.

  • ⇄Drag — position is remembered across launches.
  • ⌫Hide — right-click → "Hide Pet". Persisted to settings.
  • ⇄Switch Pet — right-click → "Switch Pet" → Claude / Lobster. Same submenu in the tray.

VIIICLAUDE CODE

Your local Claude Code sessions, in the bar.

Pick a project from ~/.claude/projects, resume any past session — or start a new one. The conversation lands in the bar like any other channel, with streaming, thinking pills, tool-call cards, and inline approval prompts. Powered by the Claude Agent SDK; drives your installed claude binary (BYO-CLI), no bundled fork.

  • ATool approval cards — Bash / Edit / Write surface inline above the composer. 1 2 3 digits, ↑↓⏎, Esc = deny.
  • BExpandable tool pills — Bash, Edit, Read, Glob, Grep, Thinking — all collapse to one line; click to see input + output.
  • CAskUserQuestion lists — model-asked option cards with the same keyboard model.
  • ⏹Real Esc abort — Stop cancels the current turn via Query.interrupt(); the SDK process stays alive for your next message.

Get ClaudeBar in your menu bar in 30 seconds.

One download, one drag. Requires the Claude Code CLI installed on your system. macOS Apple Silicon only for pre-built releases.

macOS (.dmg) v0.7.0 · arm64 All releases →

Install — macOS (Apple Silicon)

1Download ClaudeBar-0.7.0-mac-arm64.dmg.
2Open it and drag ClaudeBar into /Applications.
3First launch — right-click → Open, then "Open" in the Gatekeeper dialog. The app isn't code-signed.
4Or skip step 3: 
xattr -dr com.apple.quarantine /Applications/ClaudeBar.app

Windows

!Windows is not yet validated for v0.7.0. No pre-built release artifact. Build from source if needed.

Build from source

# clone, install, run (macOS)
git clone https://github.com/june9593/claudebar.git
cd claudebar
npm install
npm run dev:electron

# package your own build
npm run pack:mac:dmg:arm64    # macOS Apple Silicon
Frequently asked

Sensible questions, short answers.

Why isn't the build signed?

Code signing costs money — Apple Developer is $99/yr plus notarisation. ClaudeBar is a free side-project, so the DMG is unsigned. On macOS you bypass Gatekeeper once with right-click → Open (or xattr -dr com.apple.quarantine /Applications/ClaudeBar.app). One-time.

Does it work on Intel Macs?

Not in releases — we ship macOS arm64 only. The build pipeline can produce x64 (run npm run pack:mac:dmg:x64 from source), it's just not part of the GitHub Actions release. PRs welcome.

Does it work on Windows?

Windows is not yet validated for v0.7.0. The codebase compiles on Windows, but no release artifact has been tested since the ClawBar fork. Build from source and expect rough edges. Tracked as post-v0.7.0 work.

Does it phone home?

No. No analytics, no error reporting, no auto-update check. The only network calls are the ones your claude binary makes (to the Anthropic API with your own API key or Claude subscription).

Where are settings stored?

~/.claudebar/settings.json for app config (theme, window size/position, pet visibility, global shortcut, Claude CLI path). Stats cache at ~/.claudebar/usage-cache.json. SDK trace (if enabled) at ~/.claudebar/sdk-trace.jsonl. Shell-env diagnostic at ~/.claudebar/auth-debug.log.

What does ClaudeBar need to run?

An installed claude binary — install via npm install -g @anthropic-ai/claude-code or the official native installer. ClaudeBar drives your CLI via the Claude Agent SDK's pathToClaudeCodeExecutable option; the SDK's own bundled platform binary (~205 MB) is excluded from the DMG. If claude isn't found, the app shows an install guide. Tool permissions go through your existing ~/.claude/settings.json allow/deny lists, with anything not pre-approved surfacing as an inline approval card.

Is it open source?

Yes — MIT licensed. Source at github.com/june9593/claudebar. Forked from ClawBar with full git history. Issues and PRs welcome.
